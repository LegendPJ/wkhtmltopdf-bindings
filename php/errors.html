Error Troubleshooting

Apparently, there's lots of commands that could fix certain errors. This is so widespread that implementing said commands into WKPDF class is too much possibly even slowing it down. Here, you can find a list of errors and fixes kept for your convenience.

Error: "Permission Denied"

Applies: Mac OSX Linux
Reason: This is because the program is not executable. Please run the following command from some terminal/prompt (substitute OS and CPU with your system's e.g. wkhtmltopdf-lin-amd). chmod 777 wkhtmltopdf-%OS%-%CPU%

Error: "dyld: Symbol not found: __cg_jpeg_resync_to_restart"

Applies: Mac OSX
Reason: This seems due to multiple versions of libJPEG.dylib (talk about dll hell on a mac!). This means it is a failure in WKHTMLTOPDF for OSX executable. The issue is mentioned here. It involves recompiling from source.

Error: "C:\Program is not an operable..."

Applies: Windows
Reason: Windows doesn't like spaces in paths. The usual fix is to put double quotes are the path. Apparently, PHP relies on some old Windows API were te quotes fix doesn't work.
Please put your executable in a space-less path or put the pathname into PATH (and reference the executable with the filename without the path).
